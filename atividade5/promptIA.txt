        Esse código é um ótimo exemplo de laço de repetição com controle de saldo, mas notei um pequeno detalhe técnico no seu PHP: na linha $saldo=- 10;, o sinal de igual vem antes do menos, o que faz o PHP entender que você está atribuindo o valor -10 ao saldo, em vez de subtrair.

        Você gostaria que eu adicionasse um contador para mostrar quantos lanches foram comprados no total ao final da lista?

        <?php

            $tentativas = 1;

O Gemini disse

        Este código é um ótimo exemplo para ilustrar a diferença do laço do-while. Diferente do while comum, o do-while sempre executa o código pelo menos uma vez, mesmo que a condição seja falsa logo de cara (como acontece no seu código, onde 1 não é menor que 1).

        Para este cenário de "Conexão", o ideal é uma interface que simule um console de sistema ou um log de carregamento.

        Por que o resultado aparece apenas uma vez?

        No seu código original, a condição é while ($tentativas < 1). Como $tentativas já começa valendo 1, a condição é tecnicamente falsa.

            Se fosse um while comum, nada seria exibido.

            Como é um do-while, o PHP primeiro diz "Faça isso" (do) e só depois pergunta "Posso repetir?" (while).

        Pequena observação:

        Se você quisesse que ele tentasse mais vezes (por exemplo, 3 vezes), o código PHP precisaria incrementar a variável:
        PHP

        $tentativas = 1;
        do {
            echo "Tentativa $tentativas";
            $tentativas++; // Sem isso, se a condição fosse verdadeira, você teria um loop infinito!
        } while ($tentativas <= 3);
